# RSA Physiology Toolbox

This repository contains a lightweight pipeline for working with TAY Respiratory Sinus Arrhythmia (RSA) recordings collected with MindWare and processed according to `SOP_OPS227`.

The pipeline is designed to help analysts:

- organize raw and preprocessed MindWare RSA files
- import the SOP-generated MindWare HRV Excel workbook
- extract segment-level HRV/RSA metrics
- apply basic SOP QC rules
- compute additional audit and nonlinear features from exported IBI values
- generate CSV outputs and a browser-viewable feature plot report

## What The Pipeline Uses

The expected input files are:

- `.mwi`: MindWare metadata/index file; this is a SQLite database
- `.mwx`: raw MindWare signal file; must be kept with the `.mwi`
- `.edh2`: MindWare edit/history file generated after artifact correction
- `*HRV Analysis*.xlsx`: MindWare HRV workbook generated by `File > Write All Segments`

The `.mwi/.mwx` files are the raw unprocessed acquisition files. The `.edh2` file and HRV Analysis Excel workbook are generated after signal processing, manual R-peak review, artifact correction, and segment export in MindWare.

## Important Validation Note

The canonical SOP-derived HRV/RSA metrics are the values imported from the MindWare HRV Analysis Excel workbook. These are exported as:

- `mindware_hrv_stats_long.csv`
- `mindware_power_band_stats_long.csv`

The toolbox also recomputes some metrics from the exported IBI series. These recomputed metrics are useful for reproducibility checks, exploratory analyses, and QC, but they should not be treated as exact replacements for MindWare's proprietary HRV/RSA calculations.

## Expected File Naming

The pipeline searches recursively for:

- `.mwi`
- `.mwx`
- `.edh2`
- `.xlsx` files whose names contain `HRV Analysis`

It pairs files by recording stem. It also handles the observed naming difference where raw files include an extra visit/run token, for example:

```text
TAYXXXX_01_SE01_RSA.mwi
TAYXXXX_01_SE01_RSA.mwx
TAYXXXX_SE01_RSA_HRV Analysis_1_52_22 PM_23June2023.xlsx
```

## Outputs

Each recording gets its own output folder:

```text
rsa_outputs/<recording_id>/
```

For the sample data this is:

```text
rsa_outputs/TAYXXXX_SE01_RSA/
```

### `rsa_summary.txt`

A compact text summary of the run, including:

- source file information
- MindWare version/date when available
- number of analyzed segments
- number of pass-QC segments
- mean MindWare HF/RSA power
- mean recomputed HF/RSA power
- mean RMSSD
- mean sample entropy
- full-recording multiscale entropy AUC
- path to the HTML feature plots

### `mindware_hrv_stats_long.csv`

Long-format import of MindWare's HRV Stats sheet.

Typical metrics include:

- `Mean Heart Rate`
- `Mean IBI`
- `AVNN`
- `SDNN`
- `RMSSD`
- `NN50`
- `pNN50`
- `RSA`
- `# of R's Found`
- segment start/end/duration fields

These are primary SOP-derived outputs.

### `mindware_power_band_stats_long.csv`

Long-format import of MindWare's Power Band Stats sheet.

Typical metrics include:

- `LF Power`
- `LF Peak Power Frequency`
- `HF/RSA Power`
- `HF/RSA Peak Power Frequency`
- `LF/HF Ratio`

These are primary SOP-derived frequency-domain outputs.

### `rsa_segment_metrics.csv`

Transparent recomputation from the exported IBI series.

Metrics include:

- `n_ibi`
- `mean_ibi_ms`
- `mean_hr_bpm`
- `sdnn_ms`
- `rmssd_ms`
- `lf_power`
- `hf_rsa_power`
- `lf_hf_ratio`
- `hf_peak_hz`

Use this file for audit and exploratory analysis. It will not necessarily match MindWare exactly.

### `rsa_segment_qc.csv`

Segment-level QC based on available exported editing statistics.

Columns include:

- `segment`
- `max_percent_edited`
- `qc_pass`
- `qc_reason`

Current SOP rule implemented:

- fail segment if more than 10% of the segment was edited

### `rsa_nonlinear_features.csv`

Additional segment-level nonlinear IBI features:

- `cv_ibi`
- `poincare_sd1_ms`
- `poincare_sd2_ms`
- `sd1_sd2_ratio`
- `sample_entropy_m2_r02`

Segment-level entropy is exploratory because 30-second windows contain a small number of IBIs.

### `rsa_multiscale_entropy.csv`

Full-recording multiscale entropy computed from the full exported IBI sequence.

Columns include:

- `scope`
- `scale`
- `n_points`
- `sample_entropy_m2_r02`

Entropy is computed with:

- embedding dimension `m = 2`
- tolerance `r = 0.2 * SD`
- coarse-graining scales `1-10`

Blank entropy values can occur when there are too few valid template matches at a scale.

### `mwi_metadata.json`

Metadata read from the raw `.mwi` SQLite file, including:

- file information
- devices
- channel labels
- channel groups
- event timing
- packet ranges

This is useful for acquisition QC and traceability.

### `plots/feature_plots.html`

Self-contained browser-viewable plots generated without Matplotlib.

The report includes:

- mean HR by segment
- RMSSD by segment
- recomputed HF/RSA power by segment
- MindWare HF/RSA power by segment
- sample entropy by segment
- full-recording multiscale entropy curve

Open it in any browser:

```text
rsa_outputs/<recording_id>/plots/feature_plots.html
```

## QC Interpretation

For a standard SOP_OPS227 recording, expect:

- 4 minutes of recording
- 8 segments
- 30 seconds per segment
- low or zero percent edited when signal quality is good
- no segment with more than 10% edited data

The SOP also states that a segment should not be used if there are more than 3 consecutive midbeats. The current Excel workbook does not expose detailed consecutive-midbeat events, so this rule cannot yet be fully automated from the example workbook alone.

Recommended subject/session-level QC checks:

- confirm 8 expected segments are present
- confirm all or most segments pass QC
- review `max_percent_edited`
- review `% Normal Peaks` from MindWare editing stats
- inspect unusually high LF/HF ratio segments
- inspect missing entropy values at larger MSE scales
- open `feature_plots.html` and look for abrupt segment outliers

## Analysis Recommendations

For primary study analyses:

1. Use `mindware_hrv_stats_long.csv` and `mindware_power_band_stats_long.csv` as the source of SOP-derived HRV/RSA metrics.
2. Restrict analyses to segments passing `rsa_segment_qc.csv`.
3. Aggregate valid segments to subject/session-level features when needed.
4. Use recomputed and nonlinear features as exploratory or secondary features unless separately validated.

Common subject/session-level features to derive:

- mean MindWare `HF/RSA Power` across pass-QC segments
- mean MindWare `RSA` across pass-QC segments
- mean `RMSSD`
- mean `SDNN`
- mean `Mean Heart Rate`
- mean `LF/HF Ratio`
- valid segment count
- percent segments passing QC
- mean segment sample entropy
- full-recording MSE area under the curve

## Optional PhysIO Context

The AFSP project uses TAPAS PhysIO for fMRI physiological nuisance regressors from Siemens `.puls` and `.resp` files.

SOP_OPS227 RSA scoring is different. It depends on MindWare ECG R-peak review, artifact correction, and HRV segment export. PhysIO should not be treated as a drop-in replacement for the SOP MindWare workflow without validation.

The helper in `rsa_toolbox/physio.py` only writes a MATLAB path helper for future fMRI-linked work.

## Troubleshooting

### No recordings found

Check that the input folder contains `.mwi`, `.mwx`, `.edh2`, or `*HRV Analysis*.xlsx` files.

### Recording skipped

The pipeline skips recordings that do not have a MindWare HRV Analysis workbook. Run MindWare artifact correction and `Write All Segments` first.

### Entropy values are blank

Blank entropy values usually mean the segment or coarse-grained series had too few valid template matches. This is common for short 30-second segments.

### Recomputed metrics do not match MindWare

This is expected. MindWare likely uses its own handling of edited beats, interpolation, detrending, windowing, and spectral integration. Treat recomputed metrics as audit/exploratory features.

## Current Limitations

- Raw `.mwx` signal decoding is not yet implemented.
- Consecutive-midbeat QC cannot yet be fully automated without detailed edit-event export or deeper `.edh2` parsing.
- Segment-level entropy is statistically fragile because each segment has a small number of IBIs.
- Frequency-domain recomputation is not a validated clone of MindWare.

## Suggested Next Steps

- Add a one-row-per-subject/session export such as `rsa_subject_features.csv`.

- Add cohort-level QC reports once multiple participant folders are available.
